Introduction

Get started with Angular 5 boilerplate that utilize service bus architecture, that helps you to build responsive, bulletproof projects on the web with the boilerplate that is great from the smallest to atomic projects.

Quick start

Looking to quickly use that boilerplate for your project? Just clone the boilerplate from GitHub . All you need is NPM to install the modules and you are ready to go!

To start with the boilerplate, just follow the steps:

  1. Clone the boilerplate from GitHub
  2. Install node package manager or Yarn
  3. If you have NPM go to the root of the project and type in the console npm install or if you are using Yarn just type yarn
  4. When it finish, if you are using NPM type in the consolenpm run startor if you are using Yarn just type yarn start
  5. If the end of the build tells youwebpack: Compiled successfully. you are ready to go!

How translation works

To start with the translate, just follow the steps:

  1. Place our translate directive translate over the text container
  2. It will use the json translation from app/shared/translation/languages based on the current text in the container
  3. You can change the default language from English into the service.
  4. To activate the translation in a component, just fire {{ this.eventBusService.emitTranslate({}); {{ in the ngOnInit

How event-bus works

Simple example how the event bus works is our error handling:

                
                    {{
                        this.eventBusService.emitRequestError({
                            errorMessage: errorMessage
                        });
                    }}

We send a simple event via the bus and the data will be usable by just subscribing like:

                
                    {{
                        this.eventBusService.requestError.subscribe(
                            (data) => this.showError(data)
                        );
                    }}

You are able to transfer data between component without any hierarchy with much higher maintainability while keeping it easy to debug by having one source of truth

How error-handler works

When we call for something from the back-end in our services, our code usually looks like this:

                
                    {{
                    this.backendService.getSomething(arg1, arg2, ....)
                        .subscribe(
                            response => {
                                // do something on success
                            },
                            error => {
                                // do something on error

                                // ...

                                // send the error by the error handler service where it 
                                // is send to the error-handler component via the event bus 
                                this.errorHandlerService.handleRequestError(error);
                            },
                            () => {
                                // do something then
                            }
                        );
                    }}

The whole idea of the Error handler service is to controll all the ways we show to the client, that something happened by showing a single dialog in the bottom of the page to a full custom retry functionality you are free to experiment by your self by sending the correct event bus messages and expanding the 2 services

Project structure

To work with that boilerplate you need to know the basics of Angular. Here is good source to learn of the basics of Angular

  • src - Contains all of the front-end code
    • app - Contains all of the scripts
      • core - Contains all of the core services for the app
      • feedback - Contains all of the directives, components and services for user interaction for events and errors
      • services - Contains all of the services that are used for data transformation of the fetched from back-end
      • shared - Contains all of the directives, components and services that are used in multiple locations
    • assets - Contains all of the fonts, images and other assets.
      • fonts - Contains all of the fonts
      • images - Contains all of the images
    • enviroments - Contains all of the enviroment variables.
      • enviroment.local.ts - Contains all of the enviroment variables for the local development
      • enviroment.prod.ts - Contains all of the enviroment variables for the prod development
      • enviroment.ts - Contains all of the enviroment variables for when enviroment is not selected
    • styles.scss - Contains all of the global styling.
    • index.html - Contains the basic HTML page of the app.
  • package.json - Contains all of the used modules.
  • .angular-cli.json.json - Contains all of the build procedures.
